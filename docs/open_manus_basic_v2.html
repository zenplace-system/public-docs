OpenManusとは？

OpenManusは、招待制の最先端AIエージェント「Manus」の能力をオープンソースで再現するプロジェクトです。自律的に複雑なタスクを実行できる点が特徴です。

主な機能

  • マルチエージェントシステム：複数のAIエージェントが連携・協調
  • タスク自動実行：計画やデータ分析を自動化
  • ツール統合：ブラウジングやコード実行などを組込み
  • Dockerコンテナ対応：環境構築が容易
  • 拡張可能な設計：機能追加が簡単
旅行プランの作成から株式分析まで、様々なタスクを自律的に実行できます！

Mac環境でのセットアップ

必要な環境と前提条件

Docker
Docker Compose
Python 3.9+
Node.js 16+
Git

インストール手順

  1. リポジトリのクローン
    git clone https://github.com/henryalps/OpenManus.git
    cd OpenManus
  2. 設定ファイルの用意
    cp config/config.example.toml config/config.toml

    エディタでconfig.tomlを開き、APIキーなどを設定：

    [llm]
    model = "gpt-4"
    base_url = "https://api.openai.com/v1"
    api_key = "sk-..."
    max_tokens = 4096
    temperature = 0.0
    組織内で独自のモデルやプロキシを使用する場合はbase_urlやmodelを適宜変更してください。
  3. Dockerイメージのビルド
    docker-compose build
  4. コンテナの起動
    docker-compose up

    成功すると、エージェントのAPIサーバがポート5000番で待機します。

動作確認手順

CLIによる対話

コマンドラインから直接エージェントと対話できます：

python client.py --task "3日間の東京旅行の計画を立てて"

エージェントが自律的に旅行計画を立案し、結果を出力します。

REST APIによる確認

APIに直接リクエストを送って動作確認できます：

ステータス確認

curl http://localhost:5000/status

正常動作時のレスポンス：{"status":"running"}

タスク実行

curl -X POST -H "Content-Type: application/json" \
     -d '{ "task": "Hello, OpenManus!" }' \
     http://localhost:5000/task
複雑なタスクも同様に指示できます。実行に時間がかかる場合があります。

毎月の売上データを自動で分析し、トレンドや改善提案をレポートにまとめます。

データ収集
データ分析

実装サンプル

Pythonから簡単にAPIを呼び出すサンプルコード：

import requests
import json

task = "当月の売上データを分析し、主要なトレンドと改善提案をまとめてください。"
response = requests.post("http://localhost:5000/task", json={"task": task})

if response.status_code == 200:
    result = response.json().get("result")
    print("生成されたレポート:\n")
    print(result)
else:
    print(f"タスク実行リクエスト失敗: ステータスコード {response.status_code}")
社内データベースとの連携が必要な場合は、OpenManusのツール拡張機能を使うと便利です。

実行手順とポイント

  1. 環境起動：Docker環境を事前に起動（docker-compose up）
  2. タスク実行：APIリクエストでタスクを送信
  3. 結果確認：レスポンスからレポート内容を取得
タスク指示は具体的に記述すると良い結果が得られます。

トラブルシューティング

よくある問題と解決策をチェックして、スムーズに利用しましょう！

コンテナ起動問題

Dockerコンテナが起動しない場合：

  • DockerとDocker Composeのバージョンを確認
  • Appleシリコン(M1/M2)の場合、アーキテクチャ互換性を確認
  • docker-compose.ymlにplatform: linux/amd64を追記

ポート競合エラー

デフォルトポート（5000/3000）が使用中の場合：

# docker-compose.ymlでポート変更
ports:
  - "5100:5000" # ホスト側を5100に変更

APIキー関連エラー

  • APIキーが正しく設定されているか確認
  • APIキーの有効期限や使用上限をチェック
  • 社内プロキシ環境ではネットワーク設定を確認

結果の質向上

  • タスク指示を具体的に記述
  • 大きなタスクは段階的に分割して実行
